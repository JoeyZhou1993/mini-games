# Block Blast(极简单机版)

8×8 拖块消除,无尽模式,单文件零依赖。**在线玩:https://joeyzhou1993.github.io/block-blast/**

## 玩法

- 底部 3 个候选块,拖到棋盘任意空位;放一个立即补一个
- 填满一行或一列 → 消除得分;连续消除有 Combo 倍率(上限 ×5)
- 3 个块都放不下 → 死局结算。目标:刷最高分(👑 本地持久化)
- 新玩家首局有预铺引导,拖一下就能消除

## 数值设计(全部旋钮在 `index.html` 的 `CFG`)

| 旋钮 | 当前值 | 含义 |
|---|---|---|
| `stages` | [30, 110] | 发牌三阶段切换点(累计放置块数):暖场/心流/收官 |
| `redraw` | [10, 5, 0] | 各阶段发牌重抽预算——抽到"当前可放"为止,预算耗尽就裸随机(可能给你放不下的块) |
| `rescueThresh` | [0.40, 0.60, null] | 拥挤度达到阈值时强制塞一个可放小块(null=收官不救) |
| `weights` | 见代码 | 11 种块形 × 3 阶段发牌权重 |
| `clearBase` / `comboCap` | 10 / 5 | 消除基础分 / 连消倍率上限 |
| `clearAnimMs` / `overDelayMs` / `dangerOcc` | 380 / 1000 / 0.72 | 消除动画窗口 / 死局到弹窗间隔 / 危险红光阈值 |

**标定结论**(贪心 AI 模拟 500 局):中位死亡 ≈117 块 ≈ **10 分钟/局**(5秒/块),死时分数中位 ~1000,暖场期零死亡。改 `stages[1]` 控局长,改 `redraw[1]` 控中盘压力。

## 节奏设计思路

难度不靠计时器,靠**发牌三阶段收紧**:

```
0-30块   暖场: 重抽10次保底可放 + 激进救场      → 连消不断,建立信心
30-110块 心流: 重抽5次, 偶尔给放不下的块        → 需要规划空间
110块后  收官: 裸随机发牌, 大块权重拉高          → 几步内自然死局
```

## 技术要点

- **单文件** `index.html`,无构建、无依赖、无图片(方块=CSS conic-gradient 宝石切面,图标=emoji)
- **音效全合成**:Web Audio 振荡器 + 包络(软起音防爆音),消除=C大调五声音阶琶音
- 拖拽:Pointer Events(触屏偏移 2.2 格防手指遮挡,鼠标 0.7 格);落点预览+可消除行列脉冲高亮
- 健壮性:iOS AudioContext resume、localStorage try-catch(隐私模式)、rAF 节流保底、横屏旋转提示、刘海屏安全区
